Detailed Description

The main API to the JK Builder Library.

The idea of the JK Builder Library is to provide a nice, clean, easy to use way to build J and K matrices using distributed or shared architectures. This is to some extent the class that handles it (I say some extent because it is advisable and probably really good practice to derive from this class a child class that turns your objects into the input for my class, see Psi4JK for an example that turns Psi4's objects into my objects). If you are interfacing with my class your goal should be to call the SetUp function with the appropriate objects, which are the basis set, the molecule, the density matrix, the J matrix, and the K matrix (the last three are double stars for maximum compatibility)

Sets up a bare-bones, distributed, JK object in the DesiredBasis, on the desired molecule.

The JK object minimally needs three things: a basis, a molecule, and density matrix. The Carts array should be in "messy" form, which is:
<atomic number>=""> <x-coordinate> <y-coordinate> <z-coordinate>
this way we do not have to pass the identities in separately. Also note the atomic numbers will be mapped to integers via an int cast. To be on the safe side you may want to use numbers slightly bigger then the desired value, e.g. 6.01 for carbon; however simply setting it to 6.000000000000....should be OK as well. Also note that JK objects can only be used for one molecule and basis set at a time.
